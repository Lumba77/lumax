System check

HTTP probes from the Web UI bridge (same Docker network as lumax_ops). Optional rows are expected to fail when that stack is stopped (e.g. Turbo vs Chatterbox GPU switch). TTS uses the mouth bridge POST /tts (routing follows the mouth backend file / GPU stack).

This tab is the container-stack counterpart to the host AUTOTEST_Lumax_current.ps1 sentinel (Godot guardian import pass, XR launch, log tail, optional screen capture). The Web UI does not spawn Godot from here; run that script on Windows when you need XR/cache drift checks.
